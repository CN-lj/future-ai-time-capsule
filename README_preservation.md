# 保存说明

这份保存包的目标不是把信藏起来，而是让它更难消失、更容易被未来读到。

建议使用“3-2-1”保存法：

1. 至少保存 3 份副本。
2. 至少放在 2 种不同介质上，例如电脑硬盘、U 盘、移动硬盘、云盘、纸质打印。
3. 至少有 1 份放在异地，例如可信朋友处、云端账号、公开仓库或长期归档服务。

推荐的保存路径，从私密到公开：

1. 私密副本：保存在你的电脑和一个 U 盘里。
2. 云端副本：上传到网盘、邮箱附件、密码管理器附件或私人 Git 仓库。
3. 人际副本：发给一个你信任的人，请他保存。
4. 公开副本：放到 GitHub/GitLab 仓库、Gist、个人网站、博客或 Internet Archive。
5. 物理副本：打印出来，夹在书里，或放进防潮袋。

每次复制后，可以用 `SHA256SUMS.txt` 对照文件是否损坏。只要文件内容完全相同，SHA256 值就应完全相同。

最稳妥的方式不是寻找一个永不损坏的地方，而是让它在多个地方以简单格式存在：Markdown、纯文本、压缩包、纸张。越简单，越容易被未来恢复。
